# Scroll Capture — автоматический скриншот длинной статьи

Программа **перехватывает мышь**, **имитирует прокрутку колёсиком** в окне VRM/удалённого рабочего стола и **склеивает скриншоты** в один длинный PNG-файл.

Подходит, когда на удалённом ПК есть текст/документ, но нельзя скопировать или выгрузить файл за контур.

## Установка

```bash
pip install -r requirements.txt
```

Windows (рекомендуется нативный метод прокрутки `win32`, он включён по умолчанию):

```bash
python -m pip install mss numpy Pillow pyautogui pynput
```

## Быстрый старт

1. Откройте статью в VRM/браузере на удалённом ПК.
2. Прокрутите в **самое начало** статьи.
3. Запустите:

```bash
python scroll_capture.py
```

4. Наведите мышь на **левый верхний**, затем **правый нижний** угол области статьи.
5. После обратного отсчёта переключитесь в окно VRM.

Итоговый файл: `scroll_capture_ГГГГММДД_ЧЧММСС.png` в текущей папке.

## Прокрутка колёсиком

Скрипт **не использует клавиши Page Down** — только движение мыши и события колёсика.

Перед началом захвата программа **один раз кликает в центр выбранной области**, чтобы фокус попал на внутренний скроллер. Повторных кликов при прокрутке нет — это снижает риск случайного открытия картинок.

| Параметр | Описание |
|----------|----------|
| `--scroll-method win32` | Нативный Windows API `WM_MOUSEWHEEL` (по умолчанию на Windows) |
| `--scroll-method pynput` | Альтернатива через библиотеку pynput |
| `--wheel-notches 12` | Сколько «щелчков» колёсика за один шаг |
| `--micro-steps 8` | Разбить шаг на несколько коротких прокруток |
| `--focus-each-step` | Кликать в область перед каждым шагом (опционально) |
| `--scroll-delay 0.45` | Пауза после прокрутки перед снимком |

По умолчанию: `win32` на Windows, 20 щелчков колёсика, один клик при старте, пауза 0.8 сек. Достаточно:

```bash
python scroll_capture.py
```

Если страница не листается, попробуйте `--scroll-method pynput` или увеличьте `--wheel-notches`.

## Другие параметры

| Параметр | Описание |
|----------|----------|
| `--region L,T,W,H` | Область захвата в пикселях |
| `-o file.png` | Имя выходного файла |
| `--countdown 5` | Секунд до старта |
| `--max-frames 300` | Лимит кадров |
| `--save-frames ./frames` | Сохранить отдельные кадры |

## Экстренная остановка

Резко переместите мышь в **левый верхний угол экрана** (pyautogui FAILSAFE).

## Советы для VRM

- Выбирайте область точно по тексту статьи, не захватывая боковые панели.
- Если склейка «рвётся» — уменьшите `--wheel-notches`.
- Если не доходит до конца — увеличьте `--wheel-notches` или `--max-frames`.


## Качество склейки

Если таблицы перекрывают текст:
- уменьшите `--wheel-notches` (например `10`) — больше перекрытие между кадрами
- сохраните кадры для отладки: `--save-frames ./frames`
- в логе смотрите значения `перекрытие XXXpx` — они должны быть стабильными

По умолчанию используется улучшенный алгоритм выравнивания по строкам текста и границам таблиц.
