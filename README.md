# Scroll Capture — автоматический скриншот длинной статьи

Программа для Linux, которая **перехватывает мышь**, **прокручивает страницу** в окне VRM/удалённого рабочего стола и **склеивает скриншоты** в один длинный PNG-файл.

Подходит, когда на удалённом ПК есть текст/документ, но нельзя скопировать или выгрузить файл за контур.

## Требования

- Linux с графическим сеансом (X11 или XWayland)
- Python 3.10+
- Окно VRM/браузера со статьёй должно быть видно на экране

### Системные пакеты (Ubuntu/Debian)

```bash
sudo apt install python3-pip python3-tk scrot
```

`python3-tk` нужен для `pyautogui` на некоторых дистрибутивах.

## Установка

```bash
cd /path/to/scroll-capture
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Быстрый старт

1. Откройте статью в VRM/браузере на удалённом ПК.
2. Прокрутите в **самое начало** статьи.
3. Запустите программу:

```bash
python scroll_capture.py
```

4. Наведите мышь на **левый верхний**, затем **правый нижний** угол области статьи (без панелей браузера, если возможно).
5. После обратного отсчёта переключитесь в окно VRM — программа сама прокрутит страницу и сохранит PNG.

Итоговый файл: `scroll_capture_ГГГГММДД_ЧЧММСС.png` в текущей папке.

## Параметры

| Параметр | Описание |
|----------|----------|
| `--region L,T,W,H` | Область захвата в пикселях (без интерактивного выбора) |
| `-o file.png` | Имя выходного файла |
| `--countdown 5` | Секунд до старта (по умолчанию 5) |
| `--scroll-clicks 8` | Сила прокрутки за шаг (увеличьте, если листает слишком медленно) |
| `--scroll-delay 0.35` | Пауза после прокрутки перед снимком |
| `--max-frames 300` | Лимит кадров |
| `--save-frames ./frames` | Сохранить отдельные кадры для отладки |

Пример с фиксированной областью:

```bash
python scroll_capture.py --region 120,80,900,700 --scroll-clicks 10 -o article.png
```

## Экстренная остановка

- **Резко переместите мышь в левый верхний угол экрана** — сработает защита `pyautogui` (FAILSAFE).
- Или закройте терминал / `Ctrl+C` до начала захвата.

## Как это работает

1. Захватывается прямоугольная область экрана (окно статьи в VRM).
2. Курсор перемещается в центр области и крутится колёсико мыши вниз.
3. После каждой прокрутки делается скриншот.
4. Когда два кадра подряд почти совпадают — считается, что достигнут конец страницы.
5. Кадры склеиваются с учётом перекрытия (поиск совпадающих полос по пикселям).

## Советы для VRM

- Разверните окно VRM на весь экран или задайте `--region` точно по статье.
- Уберите лишние панели браузера из области захвата.
- Если склейка «рвётся» — уменьшите `--scroll-clicks` (больше перекрытие между кадрами).
- Если долго не доходит до конца — увеличьте `--scroll-clicks` или `--max-frames`.
- Медленный удалённый рабочий стол: увеличьте `--scroll-delay` до `0.6`–`1.0`.

## Ограничения

- Работает только там, где виден экран (не в чистом SSH без GUI).
- На Wayland поддержка `pyautogui` может быть ограничена; при проблемах используйте X11/XWayland.
- Программа управляет **вашей** мышью — во время работы не трогайте клавиатуру и мышь.
